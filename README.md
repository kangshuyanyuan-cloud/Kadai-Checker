# 熊本大学 Moodle 課題自動チェック・通知システム

## 概要
熊本大学のMoodleに自動でアクセスし、未提出の課題をリストアップするツールです。  
提出期限が24時間以内に迫っている課題がある場合は、パソコンの起動時にWindowsの通知機能で自動的にお知らせします。

---

## 開発の背景と思考プロセス

### 1. セキュリティの壁とAPIの不在
大学のポータルサイトで課題を確認するのが面倒で、プログラムから直接システムに入り込んで課題データを取得しようと考えました。  
しかし、大学のシステムはセキュリティが厳しく、通常のプログラムからのアクセスは弾かれてしまいます。APIも公開されておらず、途方に暮れていました。

### 2. 長期インターンでの気づき
並行して始めた長期インターンで **UiPath（パソコンの画面操作を自動化するRPAツール）** を使って業務にあたる機会がありました。  
UiPathで実際にブラウザの画面操作を自動化する仕組みを見ているうちに、  
**「人間がブラウザを操作するのと同じように振る舞えば、セキュリティの壁を突破できるのではないか？」** と気づきました。

### 3. セキュリティ突破、そして新たな壁
実際にUiPathで操作プロセスを組んでみたところ、見事にポータルサイトへのログインに成功。  
しかし、ログインできたものの **「画面から課題の情報をどうやって綺麗に抜き出すか」** が分からず、開発は再びストップしてしまいました。

### 4. AIコーディングツールとの出会い
一方、趣味でAIコーディングツールについて勉強しようとYouTubeで情報を集めていたところ、AIコーディングツールを使えば **Playwright** などのライブラリを用いてブラウザを直接操作する複雑なプログラムも構築できることを知りました。

**「UiPathで思いついた『セキュリティ突破のアイデア』を実現しつつ、中に入った後の『情報抽出の問題』も一気に解決できるのではないか？」**

この仮説をもとに、AIエージェント（Antigravity）とペアプログラミングを行いながら実装を進めた結果、Moodleから課題を自動取得し、通知を飛ばすシステムの完成に至りました。

---

## 主な機能

### 📋 手動チェック (`run_and_show.py`)
デスクトップのショートカットをダブルクリックするだけで、ターミナル上に未提出・提出済みの課題を一覧表示します。  
番号を入力すると、該当する課題ページをブラウザで直接開くことができます。

### 🔔 自動通知 (`check_and_notify.py`)
パソコン起動時にバックグラウンドで自動実行され、提出期限が24時間以内に迫っている課題があればWindowsの通知で知らせます。  
課題がない場合も「チェック完了」の通知が出るので、ちゃんと動いていることが確認できます。

### ⚙️ 自動通知の設定 (`自動通知登録.bat`)
管理者権限不要。ダブルクリックするだけでWindowsのスタートアップに通知スクリプトを登録します。

## セットアップ

### 前提条件
- Windows 10/11
- Python 3.10以上
- Google Chrome

### インストール手順

```bash
# 1. リポジトリをクローン
git clone https://github.com/YOUR_USERNAME/kumadai-moodle.git
cd kumadai-moodle

# 2. ライブラリをインストール
pip install -r requirements.txt

# 3. Playwrightのブラウザをインストール
playwright install chromium

# 4. 環境変数を設定
cp .env.example .env
# .envファイルを開いてMoodleのURLなどを設定
```

### 使い方

```bash
# 手動で課題をチェック
python run_and_show.py

# 自動通知を設定（デスクトップの「自動通知登録.bat」をダブルクリックでもOK）
# → パソコン起動時に自動で課題チェック＆通知が動くようになります
```

## 注意事項
- このツールはMoodleのカレンダーイベントから課題情報を取得しています。カレンダーに登録されていない課題は検出できません。
- 初回実行時にMoodleへのログインが必要です。2回目以降はプロファイルが保存されるため自動ログインされます。
- 個人利用を目的としたツールです。大学のシステムに過度な負荷をかけないようご注意ください。
