# 🎓 Moodle課題チェッカー

「記憶力」への依存を排除する、**課題未提出撲滅オートメーションツール**である。
AIエージェントを活用したアジャイル開発により、強固なセキュリティの壁を突破し、ブラウザ操作の完全自動化と情報抽出を実現した。

## 🛠 開発の動機
大学の課題提出プロセスには、以下の**「ヒューマンエラーを誘発する構造的な欠陥」**が存在していた。

1. **ポータル確認の漏れ**: 
   大学のシステムに自発的にアクセスしなければならないため、確認自体を忘れて課題を未提出にしてしまう。
2. **意志力への過剰依存 **: 
   「次は忘れないように手動でメモしよう」と個人の記憶力や気合に頼るアプローチでは、根本的な解決に至らない。

開発者自身が幾度となく「課題の出し忘れ」を経験した原体験から、意志力に頼るのではなく、技術の力で「絶対に忘れない仕組み」を構築し、未提出というヒューマンエラーを完全に撲滅するために本ツールを開発した。

## 🧗 乗り越えた課題と思考プロセス

本システムの開発において、最大の障壁は大学ポータルサイトの閉鎖性であった。以下の思考プロセスを経て課題を解決した。

### 1. セキュリティの壁とAPIの不在
* **課題**: 当初はプログラムから直接システムにアクセスし、課題データを取得しようと試みた。しかし、大学のシステムはAPIが公開されておらず、通常のプログラムによるアクセスは強固なセキュリティで弾かれてしまう状態であった。

### 2. RPA（UiPath）での気づき
* **思考プロセス**: 並行して始めた長期インターンでRPAツール「UiPath」を用いて業務にあたる機会があり、ブラウザ操作を自動化する仕組みを観察。ここで「人間がブラウザを操作するのと同じように振る舞えば、セキュリティの壁を突破できるのではないか？」という仮説を得た。しかし、ログイン後の「画面から課題情報をどうやって綺麗に抜き出すか」の解が見つからず、開発は一時ストップした。

### 3. AIツールとつながる瞬間
* **思考プロセス**: その後、趣味でAIコーディングの情報を集めるためYouTubeを見ていたところ、AI開発環境「Antigravity」がChromeブラウザを直接操作できることを知る。
* **ブレイクスルー**: この情報に触れた瞬間、「インターンで培った『RPAの要領でセキュリティを突破するアプローチ』と、Antigravityが持つ『AIによる高度なスクレイピング機能』を組み合わせれば、中に入った後の情報抽出の問題も一気に解決できるのではないか？」という仮説が成立した。この着想をもとに開発を再開し、ボット検知を回避しながら未提出課題のデータを完全に自動取得する仕組みを構築した。

## ✨ 主な機能 

### 🔔 1. 期限前強制アラート
- **ロジック**: PC起動時に自動でバックグラウンド実行され、提出期限が24時間以内に迫っている課題をWindowsのネイティブ通知で警告する。
- **UX**: 「わざわざ見に行く」という能動的なアクションを不要にし、システム側から強制的に気づかせることで確認漏れを防ぐ。

### 📋 2. ワンクリック状態確認 
- ショートカット（`run_and_show.py`）を起動するだけで、ターミナル上に未提出・提出済みの課題を一覧表示する。
- 番号を入力するだけで該当する課題ページへ直接ブラウザ遷移させ、確認から提出までの摩擦を最小化する。

### ⚙️ 3. ゼロ・コンフィグ導入
- 管理者権限不要。付属のバッチファイル（`自動通知登録.bat`）をダブルクリックするだけで、Windowsのスタートアップへの登録が完了する。

## 💻 技術スタック
本プロジェクトは**AI駆動開発 (AI-Driven Development)** の実践として、コード記述の大部分をAIで効率化している。

- **Development Tool**: Google Antigravity (AI-Integrated IDE)
- **Language**: Python 3.12
- **Automation/Scraping**: Playwright (ボット検知回避機能実装)
- **OS Integration**: Windows Forms NotifyIcon, PowerShell, VBScript (スタートアップ自動起動)
